Adaptabilidad

La Web es (casi) totalmente fluida por diseño

Eso nos enseña el proyecto Fluidity, generalmente es el desarrollador quién la rompe.

Componentes de una Web Apdatable

  • Uso de media querys: especificación + ejemplos
  • Grillas fluidas: ZURB, Bootstrap, PicniCSS, PureCSS o Caseras
  • Imágenes adaptables: Picturefill 2.0 y Lazysizes
  • Tipografía adaptables: Rem, balanceando el texto o FlowType.js
  • Tablas adaptables: Únicamente con CSS, Stackable.js o Tablesaw

Conceptos sobre performance

“Page weight isn't the only measure; focus on perceived performance”

— Scott Jehl, 2014

La performance en parte es un atributo que se percibe.

“Expect RWD to add ~10% [...] If you dig in, you can see that most of my rendering and latency problems are being caused by classic modern day web design performance [...]”

— Dave Rupert, 2014

Nuevamente es el desarrollador quién rompe las cosas.

Componentes de una web performante

  • Poner en línea el CSS Above the Fold : Técnicas
  • Miniminzar, comprimir y cachear (Imágenes, CSS y Scripts)
  • Optimizar imágenes : Compressor
  • Agilizar WebFonts : Tutorial
  • Diferir la ejecución de JavaScript : ¿Puedo usarlo ya?

Herramientas de performance

  • Google PageSpeed: Medición del rendimiento más consejos de optimización
  • Web Page Test: Análisis muy completo incluyendo líneas de tiempo
  • Perfmap: Muestra cuáles son los componentes que más demoran en cargar
